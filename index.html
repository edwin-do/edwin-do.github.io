May 2022 - Aug 2022
Software Developer Intern @ MenloLab

This was my first startup experience. We had a small but close development team, and communicated majority of our daily updates through standup. I was onboarded through a in-house CLI tool that was used to pull and spin up all the required Docker images. I started my term with a couple light tickets such as adding skeleton loaders, updating outdated packages and adding simple components in Vue.js. Afterwards, I quickly moved on to other new features such as pulling and displaying git information for a repository page, creating a file tree interface where users can navigate, add and delete files and folders, and implementing a tool bar used to zoom within the interface. ...

In the later half of my term, I was tasked to create a Desktop application that can be installed on Windows and Linux machines. I started out by researching the available options and compiled a table that compares the strengths and weaknesses between each option. The team selected a couple potential candidates, and I quickly created a mock-up for each to see its compatability and identify any early problems. After presenting my findings to the team, we decided to move forward with Tauri and I started developing the application. This application needed to communicate with sockets (linux) or named pipes (windows) using interprocess communication to determine the state of certain visual components. For example, if a connection is established, a status icon will be toggled green to reflect its state. During my testing with this application, I ran into issues arised from a race condition when refreshing the statuses of multiple networks. I was able to solve this concurrency issue by using locks and some help from a senior developer. I definitely learned a lot during this internship and would love to give everyone at MenloLab a shoutout for helping me along the way.
